MCPHub with Windsurf

July 03, 2025
#mcp #tech

I came across mcphub today and got the motivation to try it out... I use AI tools like Windsurf and Roo across multiple devices and it'd be awesome to have a simple mcp config - ie. a centralized hub to serve all my mcp servers and BAM enter this github project. It was easy to spin up in docker, and then according to Windsurf's docs here the setup is just naming a server and passing a serverUrl








        

{
  "mcpServers": {
    "hub": {
      "serverUrl": "http://localhost:3000/sse"
    }
  }
}

Warning

The sse endpoint stuff is getting deprecated eventually - I didn't want to spend any time right now trying to get the newer things to work... future me/you will solve that problem

Setup

I ran this in compose with this file








        

services:
  mcphub:
    image: samanhappy/mcphub
    ports:
      - "3000:3000"
    volumes:
      - ./mcp_settings.json:/app/mcp_settings.json
      - ./data:/app/data
  qdrant:
    image: qdrant/qdrant
    container_name: qdrant
    restart: unless-stopped
    ports:
      - "6333:6333"
    deploy:
      resources:
        limits:
          memory: "500m"
